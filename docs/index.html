Inter
Резервный шрифт. Body-текст в плотных интерфейсах, мелкие подписи, табличные данные. Не использовать в заголовках и постерах.

Один шрифт — Inter. Везде: в Google Slides, на сайтах, в email-шаблонах, в документации, в постерах, в интерфейсах.

Inter — нейтральный grotesque с сильной кириллицей и лучшей читаемостью на экране среди свободных аналогов. Лицензия OFL, доступен одной строкой в Google Slides и Google Docs. Это значит: ни один документ, который мы передаём наружу, не теряет шрифт при открытии у партнёра. Slides не подменяет его на Arial при экспорте.

Чего не делаем. Не используем Pragmatica Next в материалах, которые передаются наружу — у партнёра шрифта нет, ляжет на Times. Внутренние Figma-файлы — оставляем как есть. Не миксуем Inter с другим основным шрифтом «для разнообразия» — этого хватает.

Weights

400 для body, 500 для подзаголовков, 600 для заголовков, 300 для крупных дисплейных надписей с разреженным эффектом, 700–800 для редких силовых акцентов.

Подключение

Google Slides / Google Docs. Insert → Font → More fonts → ищем «Inter» → Add. После этого ставим как Theme font на весь шаблон.

Web. Через Google Fonts:

<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700;800&display=swap" rel="stylesheet">

font-family: 'Inter', sans-serif;

Self-hosting. Файлы — github.com/google/fonts/tree/main/ofl/inter. Variable TTF: Inter[opsz,wght].ttf. Лицензия OFL — можно класть в репозиторий.

У Radugadesign нет одного фирменного цвета. Цитата из guideline: «есть цветовая палитра, которую можно использовать при работе с материалами, где она необходима — например, в 3D-графике, иллюстрациях или мерче». Это намеренное решение бренда: цвет — инструмент, а не идентичность.

Идентичность держится на типографике (Inter) и принципах композиции. Цвет — поверх.

Структура палитры

13 hue-колонок × 6 tone-рядов = 78 цветов. Колонки идут по спектру: cyan → sky → blue → indigo → purple → magenta → pink → red → coral → orange → yellow → lime → green. Тоны — от насыщенно-светлого (bright) до угольно-тёмного (charcoal).

Как выбирать цвет

Для одного материала (постер, обложка, инсталляция) — одна-две колонки палитры. Не смешиваем три и больше hue в одной композиции, иначе теряется системность.

Контрастные пары — между крайними тонами одной колонки: bright поверх charcoal той же колонки или наоборот. Это гарантирует AA-контраст для текста.

Соседние колонки (cyan + sky, orange + yellow) — для мягких градиентов и переливов. Колонки с противоположных концов (cyan + red, blue + orange) — для резких контрастных сцен.

Чего не делаем

Не используем нейтральные серые из старых проектов как «бренд-цвет». В файле много серых из конкретных постеров и кейсов — это атавизмы, не палитра. Если нужен нейтральный — берём ink #000 или paper #FFF.

Не задаём один доминирующий цвет на всю коммуникацию бренда. Палитра — инструмент с переменной частью; постоянная часть — типографика и сетка.

SMM — форматы и шаблоны

Финальные форматы из Figma rd4-C / SMM fin. Афиши, карусели, вакансии, видео, open call, YouTube заглушки.

Все форматы финализированы в Figma rd4-C → страница SMM fin.

Форматы и шаблоны

Афиша события с человеком. Фото спикера занимает большую часть кадра. Структура: фото → имя + роль → заголовок события → место + дата. Фото всегда реальное, не иллюстрация. Формат 1:1 и 4:5.

Афиша события без фото. Монохромный или палитровый фон, крупный заголовок, место + дата. Используется когда нет фото спикера или событие коллективное.

Типовой макет статика. Универсальный шаблон: применяется для событий, open call, анонсов. Заголовок занимает верхние 2/3, мета-блок (дата, место) — нижняя полоса.

Типовой макет анимация. Та же структура, что статика. Анимация — плавное появление текстовых блоков или лёгкий parallax фото. Не более 3–5 секунд, без избыточного движения.

Вакансии. Название роли крупно, список задач коротко (5–7 строк), контакт — hr@rdgdsgn.com. Лого r обязательно. Тон — прямой, без корпоративных клише.

Видео формат. Логотип r идёт вотермаркой на всём видео (угол: правый верхний или нижний). Не делать белый текст + лого на чёрном фоне — избегать повторения. Субтитры только на сильных фразах.

Open call. Название конкурса + организатор + год. Без лишних элементов — максимально читаемо за 2 секунды.

Типовой пост с карточками (карусель). Серия 4–9 изображений: - 1-я карточка — оригинальное фото или стоп-кадр из проекта - 2-я — название проекта крупно - 3-я — короткое описание + лого r (обязательно; можно совмещать с иконкой единорога) - Остальные — цитата / интервью / факты о проекте

Адаптация: 1:1 для ленты, 4:5 вертикаль для карусели, 9:16 для сторис.

Пост с карточками кросс-промо. Карусель с чужим контентом (интервью SRSLY, партнёрские публикации). Структура идентична типовому, но заголовок указывает источник.

Не типовой пост в стилистике RD. Гиды, списки мест, подборки. Нумерованные карточки с иконкой-номером. Фон — монохромный тон из палитры или заблюренный генеративным алгоритмом. Без строгой сетки, больше воздуха между элементами.

Заглушки YouTube. Два варианта: с фото человека и без. Правило из афиш применяется напрямую. Текст крупный, логотип в угол.

Спецпроект. Свободный формат, разрабатывается под конкретный проект. Базовые правила (Inter, 1–2 hue палитры) обязательны; сетка и масштаб — под задачу.

Оформление соцсетей. Шапка профиля, highlights-обложки. Используется один акцентный цвет из палитры, лого r на highlights.

Афиши: печатный и диджитал формат. Два отдельных шаблона. Печатный — A4 и A3, рассчитан на офсет (300 dpi, CMYK). Диджитал — 1920×1080 или форматы соцсетей, RGB.

Telegram-контент

Структура поста по проекту: 6–9 изображений. - Карточки-разбивки с названием проекта: монохромный фон или алгоритмический блюр + короткий текст о проекте + лого. - Не делать белый текст на чёрном фоне — уже перегрето, избегать.

Reels и видео

Лого буквой r — вотермарк на всё видео, не убирается. Позиция: правый нижний угол, небольшой отступ. Прозрачность: 70–80%, не мешает кадру.

Запрещено в SMM

Не миксовать 3+ hue в одной карточке. Не использовать шрифт кроме Inter. Не делать чёрный фон + белый текст как базовый стиль (только как исключение для конкретного проекта). Не вставлять лого полным словом «radugadesign» там, где достаточно r.

Иконки

Системный набор Icons + брендовые иконки Unicorn+. Figma rd4-C → страница icons.

Иконографика Radugadesign зафиксирована в Figma rd4-C → страница icons. Две группы: Icons (системные) и Unicorn+ (брендовые).

Icons — системный набор

Все иконки линейные, монохромные, одна толщина stroke. Стиль минималистичный, без теней и скевоморфизма.

Навигация и указатели - Стрелки во всех направлениях (↑ ↓ ← → и диагонали) - Несколько типажей буллетов (точка, тире, квадрат, галочка) - Пин / закреп - Ссылка

Инструменты - Ручка (редактирование) - Загрузка - Лампочка (идея) - FAQ

Медиа и коммуникация - Аудио: наушники, микрофон, плейбар - Камера / объект - Глаз

Время и организация - Календарь - Колокольчик / уведомление - Папка - Закладка

Реакции - Лайк / сердце - Звёздочка - Огонёк

Информационные - Молния (срочно / энергия) - Цифры (нумерация) - Нет заряда - Человек - Планета

Unicorn+ — брендовые иконки

Единорог — символ radugadesign («Unicorns of freedom since 2008»). Используется как брендовый знак, не как декоративный элемент.

Правила применения Unicorn+: - В карточках карусели — можно совмещать с лого r - Не использовать на каждом материале — только там, где нужна брендовая идентификация - Размер пропорционален логотипу r, не крупнее

Правила применения иконок

Иконки не смешиваются со сложными иллюстрациями в одном макете — либо иконки, либо фото/графика.

Цвет иконки: #000 (ink) на светлом фоне или #FFF (paper) на тёмном. Иконки из палитры — только если цвет несёт смысловую нагрузку (например, иконка-статус).

Минимальный tap-target для интерактивных иконок — 44×44 px.

Инструкция для агента

llm/system.md — raw markdown

Этот файл — system prompt для LLM-агента, который генерирует материалы в дизайн-системе Radugadesign. Загрузите его как system message; токены (tokens/typography.yaml, tokens/colors.yaml) и принципы (principles/*.md) подгружайте как reference, когда они нужны для конкретной задачи.

Идентичность

Вы — дизайн-агент бренда Radugadesign. Ваша задача — создавать материалы (постеры, презентации, посты, веб-страницы, обложки), которые опознаются как сделанные в стиле Radugadesign.

Идентичность бренда держится на трёх осях:

  1. Типографика — постоянная часть. Один шрифт Inter везде — заголовки, body, интерфейсы, мелкие подписи. Других шрифтов в материалах для внешней коммуникации нет.
  2. Композиция — постоянная часть. Чёткая сетка, крупные акцентные элементы, много воздуха.
  3. Цвет — переменная часть. У бренда нет одного фирменного цвета. Палитра — инструмент с 78 цветами в матрице 13 hue × 6 tone. Один материал использует 1–2 hue.

Жёсткие правила

Шрифты

  • Один шрифт на всё — Inter. Не миксовать с другим основным.
  • Заголовки и дисплейные надписи — Inter weight 500–600.
  • Body — Inter weight 400.
  • Дисплейные крупные надписи с разреженным эффектом — Inter weight 300.
  • Никогда не использовать Pragmatica Next в материалах для внешней передачи: у получателя её не будет, текст развалится.
  • В Google Slides и Google Docs выставлять Inter как Theme font.

Цвет

  • В одной композиции — не более двух hue из палитры. Одна-две колонки tokens/colors.yaml#hues.
  • Контрастные пары — между крайним светлым (bright) и крайним тёмным (charcoal) одной hue. Это даёт AA-контраст для текста.
  • Соседние hue (cyan + sky, orange + yellow) — мягкие переливы.
  • Противоположные hue (cyan + red, blue + orange) — резкие контрасты для драматичных сцен.
  • Нейтральные — только #000 (ink) и #FFF (paper). Серых нет.
  • Никаких цветов вне палитры из tokens/colors.yaml.

Композиция

  • Сетка обязательна. Элементы привязываются к колонкам, не плавают.
  • Заголовок крупный, занимает 1/3 высоты или больше для постеров и обложек.
  • Нет кучи мелких декоративных элементов: один-два акцента на материал.

Тон голоса

Прямой, без воды. Конкретные факты, не прилагательные. Не «инновационный продукт», а «инсталляция с 9 проекторами». Никаких слов «уникальный», «передовой», «синергия».

Цитата из guideline для калибровки: «ROTOR — аудиовизуальная инсталляция, исследующая распространение волн в пространстве».

Что использовать как источник правды

Перед генерацией всегда читайте свежие файлы из этого репозитория, не полагайтесь на запомненные значения:

  • tokens/typography.yaml — шрифты, веса, инструкции по подключению.
  • tokens/colors.yaml — палитра в виде матрицы hues[name][tone_index].
  • principles/typography.md — расширенные правила по типографике.
  • principles/colors.md — расширенные правила по цвету.

Если в задаче встречается что-то, что не покрыто этими файлами (анимация, иконография, тон конкретного канала) — спросите автора, не додумывайте.

Чего никогда не делаете

  • Не вводите новые шрифты «потому что подходит по настроению».
  • Не миксуете 3+ hue в одной композиции.
  • Не используете нейтральные серые из палитры старых проектов.
  • Не делаете градиенты от случайной пары цветов — только из той же hue или соседней.
  • Не подменяете Inter на Helvetica/Arial при генерации HTML/CSS — подключайте через Google Fonts.

Формат ответа

Когда задача — сгенерировать материал в виде кода (HTML, SVG, JSX, marp/pptx):

  • Указывайте tokens/colors.yaml#hues.<name>[<index>] рядом с каждым цветом, чтобы можно было трассировать решение.
  • В CSS подключайте Google Fonts через <link>, не embed.
  • Цвета выводите в верхнем регистре HEX (#1DEEEB), как в токенах.
